## Durable Architecture Facts

**Source shape**
- Entry: `tiny-world-builder.html` â€” boot config, DOM scaffold, ordered `<script src>` tags
- Styles: `styles/tiny-world.css` (~5.2k lines)
- Modules: `engine/world/00-*.js` through `69-*.js` + `99-late-boot.js` (~40k LoC total)
- All engine modules are **plain classic scripts sharing one global scope** â€” load order matters; every top-level `const`/`let`/`function` must be unique across all files. A duplicate identifier throws `SyntaxError` and silently kills that module while others continue loading. Prefix module-local globals (e.g. `_sr`, `_fl`, `_la`). New modules expose via `window.__*` from an IIFE.
- `tools/check.js` (run via `npm test`) fails the build on duplicate top-level identifiers across `engine/world/*.js`
- Three.js **r128** pinned; vendor self-hosted under `vendor/three/`. Do not bump the version.
- No bundler, no npm runtime dependencies. Static checks: `npm test`. Dist: `npm run build` or `./publish.sh`

**World data model**
- `world[x][z]` â€” intent: `{ terrain, terrainFloors, kind, floors }`
- `cellMeshes['x,z']` â€” render: `{ tile: Group, object: Group|null }`
- Always mutate via `setCell(x, z, opts)` â€” never write `world[x][z]` directly outside init
- `userData.landing` guards drop-in animations â€” never remove these checks

**Materials**
- `M.*` materials are shared across many meshes â€” clone before mutating `.color`
- `disposeGroup(group)` disposes geometries but NOT materials; cloned per-instance materials (smoke) self-dispose on death

**Cameras**
- `orthoCam`, `softCam`, `persCam` coexist; `camera` is a swappable reference
- `togglePerspective()` / `setCameraMode()` write to all projections/positions

**LandscapeEngine**
- `LandscapeEngine.js` is a superseded monolith â€” `getHeight`/chunk-building live in `engine/landscape/*.js` mixins that override it. Edit the mixins, not the monolith.

**Build pipeline**
- Dev: `npm run dev` (:3000) serves source directly â€” no build step during iteration
- Prod: `./publish.sh` â†’ `dist/` â†’ Netlify auto-deploys; `vercel.json` + `netlify.toml` both point to `dist/`
- PartyKit server (`party/index.js`) deploys via `partykit deploy`, NOT `publish.sh`

**Economy â€” two currency rails, never conflate**
- **Allowance GOLD** â€” holdings-based, non-transferable weekly allowance; `gold_ledger_events` table; not a financial instrument
- **Earned GOLD (Coins)** â€” player-to-player transferable balance earned via template sales, referrals, resource sell; `coin_balances`/`coin_ledger` tables; `lib/coins.mjs` credit/debit/transfer primitive (EC1, adversarially reviewed, advisory-lock race-safe)

**Economy gate architecture**
- `requireTinyverseAccess(auth.user, origin)` in `lib/tinyverse-access.mjs` guards all money surfaces after auth
- Allowlist widened without redeploy via `TINYVERSE_ACCESS_EMAILS` env var (comma-separated)
- **Exception:** `/api/admin/gold-payout` is gated by `x-admin-secret` only â€” intentional admin-only path, the tinyverse gate is NOT applied here (confirmed by Codex spec validation today; this is by design)
- **Exception:** `stripe-webhook` gated by Stripe signature verification only

**Multiplayer message routing trap**
