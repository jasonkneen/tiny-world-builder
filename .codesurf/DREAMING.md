## Project Overview

**tinyworld** is a single-file browser app (`tiny-world-builder.html`) — an isometric world-builder using Three.js r128 (self-hosted under `vendor/three/`). The codebase is intentionally bundler-free and has grown to approximately 29k LoC. Static output goes to `dist/` via `publish.sh`; both Vercel (`vercel.json`) and Netlify (`netlify.toml`) deploy from `dist/`.

## Durable Architecture Facts

**Core data contract (do not break)**
- `world[x][z]` — intent layer: `{ terrain, terrainFloors, kind, floors }`
- `cellMeshes['x,z']` — render layer: `{ tile: Group, object: Group|null }`
- All mutations go through `setCell(x, z, opts)` — writing directly to `world[x][z]` outside of init desyncs intent from rendering
- Far-flung ghost board user overrides also live in `world[gx][gz]` at global coords (no separate override map)

**World schema (world.schema.json)**
- Transform array accepts 4-item prefix: `[x, y, z, offsetY]` — `offsetY` persisted and restored
- Appearance properties: `scaleX`, `scaleY`, `scaleZ`, `materialTexture` + scales, `bodyTexture`, `topTexture` + scales

**Three.js constraints**
- r128 is pinned; do not bump — shadows and material color spaces differ in newer releases
- Materials in `M.*` are shared; clone before mutating color
- `disposeGroup()` disposes geometries but not shared materials — per-particle smoke clones + disposes on death; follow that pattern for unique-material instances
- Three active cameras: `orthoCam`, `softCam`, `persCam`; `camera` is a live reference swapped by `togglePerspective()` / `setCameraMode()`
- `customTextureMaterial(base, uvScale)` — cached clone helper; use for per-material UV scale; do not mutate shared materials directly

**Selection system (multi-target, current)**
- `selectedBoardObjectTargets` replaces old single-target helpers
- `updateSelectedBoardObjects()`, `scaleSelectedBoardObject(axis, delta)`, `moveSelectedBoardObject(dy)` operate on all selected targets
- `mesh.userData.objectScaleBaseVec` stores base scale vector; landing/fade animations honour it
- Selection pane exposes: per-axis scale, offsetY, material assignment, material scale, terrain/height quick controls
- `SURFACE_LINKED_MODEL_SCALES` and `linkedSurfaceMaterialTextureScale` for per-material scale overrides

**Style**
- Semicolons required, 2-space indent, single quotes, trailing commas where present
- Section headers: `// -------- name --------` — keep related code grouped

**Performance**
- Grid range: 8×8 default up to 48×48 via settings
- At larger grids: preserve progressive rendering, avoid broad synchronous rebuilds
- Clouds: reuse dummy objects, cache particles

## Ghost World Generation Notes

- `makeGhostWorld(boardX, boardZ)` must be deterministic; cached in `ghostBoardCells['bx,bz']`
- RNG: `ghostHash` for board-level; per-cell `cellRand` always uses **global** coords — never local
- Path/river: horizontal Z = `pathZForRow(boardZ)` only; vertical X = `pathXForCol(boardX)` only — never couple both axes
- Bridges auto-inserted where river crosses horizontal path (`kind: 'bridge'`)
- Cell schema must include `{ terrain, kind, floors, buildingType, fenceSide, extras }` — omitting `extras: []` caused subtle bugs
- User overrides in global `world[][]`; `ghostCellAt` prefers them; `buildGhostBoard` skips cells owned by `setCell`
- `ghostBoardsBlank = true` → blank grass everywhere outside home board
